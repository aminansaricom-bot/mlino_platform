# Reviews — sprint-01

One file per reviewed Pull Request: `PR-<number>-review.md`.

## Convention

Each review file should contain:
- PR number and title
- Engineer / task it closes
- Review verdict: Approved / Changes Requested / Rejected
- Architecture notes
- Code quality notes
- Anything that became a new Issue, ADR, or ROADMAP entry as a result

Reviews are logged here **in addition to** the actual GitHub PR review
(this is the durable, searchable record; GitHub's own review UI is the
live one). No PRs have been opened yet this sprint — this directory is
empty until the first one lands.
